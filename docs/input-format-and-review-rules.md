# 입력 포맷 및 검토 규칙

## 1. 목적

월초청구서 자동화는 사용자가 월별 폴더에 넣은 자료와 근태 메모를 결합해 전자결재 입력 후보를 만든다. 자동화는 청구 가능성이 높은 항목을 웹브라우저에 입력하고, 출퇴근/개인 이동 가능성이 있는 항목은 미리보기에서 제외 또는 검토 대상으로 표시한다.

## 2. 입력 자료

| 자료 | 위치 | Git 관리 |
| --- | --- | --- |
| 교통사용 내역 | 월별 NAS 폴더 | 제외 |
| 영수증 이미지 | 월별 NAS 폴더 | 제외 |
| 야근식대 텍스트 | 월별 NAS 폴더 | 제외 |
| 집/본사 정류장 설정 | `config/commute-rules.local.ini` | 제외 |
| 근태 메모 | `local/attendance/YYYY-MM.md` | 제외 |
| 예시 설정/예시 메모 | `config/*.example.*` | 포함 |

## 3. 출퇴근 제외 규칙

사용자가 설정한 집 정류장과 본사 정류장을 기준으로 양방향 이동은 정기 출퇴근으로 판단한다.

```ini
[commute]
home_stop = HOME_STATION_NAME
office_stop = OFFICE_STATION_NAME
exclude_routes =
  HOME_STATION_NAME -> OFFICE_STATION_NAME
  OFFICE_STATION_NAME -> HOME_STATION_NAME
```

처리 방식:

- 출퇴근 제외 구간은 `excluded_commute`로 표시한다.
- `excluded_commute` 항목은 전자결재 웹브라우저에 자동 입력하지 않는다.
- 미리보기에는 제외 사유와 함께 남겨 사용자가 확인할 수 있게 한다.
- 정류장명이 변경되면 local ini 파일만 수정한다.

## 4. 교통비 청구 후보 규칙

| 조건 | 상태 | 웹브라우저 입력 |
| --- | --- | --- |
| 출퇴근 제외 구간과 정확히 일치 | `excluded_commute` | 입력 안 함 |
| 외근일의 출퇴근 제외 구간 외 이동 | `claimable` | 입력 |
| 평일이고 출퇴근 제외 구간이 아님 | `claimable` | 입력 |
| 주말/공휴일 이동 | `review_required` | 기본 입력 안 함 |
| 연차/반차일 이동 | `review_required` | 기본 입력 안 함 |
| 출발/도착 정류장 파싱 실패 | `review_required` | 기본 입력 안 함 |

주말이나 개인적인 이동은 자동화가 완전히 판단하기 어렵다. 따라서 `review_required`로 표시하고, 사용자가 미리보기에서 삭제하거나 승인한 뒤 입력한다.

## 5. 상세내역 작성 규칙

### 외근 교통비

전자결재 `상세내역`에는 교통사용 내역의 승하차 내용을 적는다.

예시:

```text
출발정류장-도착정류장
```

근태 메모에 외근지가 있으면 검증용 메모로 사용한다.

예시 내부 모델:

```json
{
  "expense_date": "2026-06-25",
  "source_type": "transport",
  "attendance_type": "offsite",
  "work_location": "고객사C",
  "detail": "출발정류장-도착정류장",
  "review_status": "claimable"
}
```

### 야근식대

전자결재 `상세내역`에는 근태 메모의 퇴근 시간을 적는다.

예시:

```text
20:38
```

근태 메모에서 해당 날짜의 퇴근 시간이 없으면 `review_required`로 표시한다.

## 6. 근태 메모 파싱 규칙

사용자는 아래처럼 간단한 텍스트를 작성한다.

```text
1일 20시30분
2일 연차
3일 외근 고객사A
4일 오전 반차
5일 21시10분
```

파싱 결과:

| 원문 | 결과 |
| --- | --- |
| `1일 20시30분` | 야근 퇴근 시간 `20:30` |
| `2일 연차` | `leave_full_day` |
| `3일 외근 고객사A` | `offsite`, 외근지 `고객사A` |
| `4일 오전 반차` | `leave_half_day_am` |

## 7. 출력 미리보기

자동 입력 전 `claim-preview.md`에는 최소한 아래 구분을 표시한다.

```text
## 전자결재 입력 대상
- 2026-06-25 / 업무교통비 / 4,450원 / 출발정류장-도착정류장

## 입력 제외
- 2026-06-05 / 업무교통비 / 1,500원 / HOME_STATION_NAME-OFFICE_STATION_NAME / 출퇴근 제외 구간

## 사용자 검토 필요
- 2026-06-08 / 업무교통비 / 1,500원 / 이동구간 / 연차일 이동
```

브라우저 자동 입력은 `전자결재 입력 대상`만 수행한다.
