# Codex 보조 월초청구서 처리 운영 절차

## 1. 목적

월초청구서 개인경비 처리는 완전 자동 Python 프로그램만으로 해결하기보다, 사용자가 월별 자료를 정리하고 Codex가 반복 입력과 검증을 보조하는 방식으로 운영한다.

목표는 다음과 같다.

- 사용자가 직접 입력해야 하는 값을 최소화한다.
- 월별로 달라지는 프로젝트, 근태, 영수증, 예외 항목은 사용자가 짧은 메모로 제공한다.
- Codex는 폴더, 엑셀, 근태 메모, Chrome 전자결재 화면을 확인해 입력 후보를 만든다.
- 결재요청, 상신취소, 삭제처럼 외부 상태를 확정 변경하는 동작은 사용자가 직접 수행한다.

## 2. 역할 분리

### 2.1 사용자가 수동으로 처리할 일

사용자는 매월 아래 자료를 월초청구서 폴더에 준비한다.

```text
D:\02_nas\onycom\01_월초청구서\YYYY년M월 월초청구서
```

필수 준비 항목:

- `교통사용_내역.xlsx`
- 필요 시 원본 `교통사용_내역.xls`
- 야근식대 영수증 이미지
- 외근 식대 영수증 이미지
- 택시 영수증 이미지
- 근태/외근/야근 메모

사용자 처리 기준:

- 청구하지 않을 교통사용 내역은 엑셀에서 취소선으로 표시한다.
- 집과 본사 사이의 정기 출퇴근 구간은 local 설정 또는 엑셀 취소선으로 제외한다.
- 주말 이동은 근태 메모에 주말출근 또는 휴일근무 근거가 있는 경우만 청구 후보로 둔다.
- 야근식대는 해당 날짜의 퇴근 시간을 근태 메모에 적는다.
- 외근 교통비는 외근일과 고객사 방문 여부를 근태 메모에 적는다.
- 전자결재 신규 지출결의서 화면은 사용자가 Chrome에서 로그인한 상태로 열어둔다.
- Codex가 입력한 후 최종 문서 내용, 합계, 첨부파일, 결재선을 사용자가 확인한다.
- 결재요청은 사용자가 직접 클릭한다.

권장 파일명:

```text
01_근태확인.txt
교통사용_내역.xlsx
교통사용_내역.xls
야근 식대 {일}일.jpg
외근 식대 {일}일.jpg
교통사용_내역_택시{번호}.jpg
```

### 2.2 Codex가 자동 또는 반자동으로 처리할 일

Codex는 사용자가 준비한 자료를 기준으로 아래 작업을 수행한다.

- 월별 폴더의 파일 목록을 확인한다.
- `교통사용_내역.xlsx`를 우선 읽고, 취소선이 있는 행을 입력 대상에서 제외한다.
- `01_근태확인.txt` 또는 사용자가 제공한 근태 메모를 읽어 야근, 외근, 연차, 반차, 주말출근 여부를 분류한다.
- 출퇴근 제외 구간, 취소선 제외, 주말 근태 없음, 연차/반차 이동을 입력 제외 또는 검토 대상으로 분리한다.
- 자동 입력 대상, 입력 제외, 사용자 검토 필요 항목을 미리보기로 작성한다.
- 사용자가 확인한 뒤 Chrome 전자결재 화면에 입력 대상만 입력한다.
- 첨부 대상 파일 목록을 보여주고, 사용자 확인 후 첨부를 보조한다.
- 사용자가 결재요청을 완료한 뒤, 상신 상태, 현재 결재자, 첨부 개수, 합계를 읽기 전용으로 확인한다.

Codex가 직접 수행하지 않을 일:

- 그룹웨어 로그인 자동화
- 결재요청 클릭
- 상신취소 클릭
- 결재선 임의 변경
- 원본 영수증 또는 교통사용 내역을 Git에 커밋
- 쿠키, 토큰, 비밀번호, 세션 파일 저장

## 3. 처리 파이프라인

### 3.1 입력 준비 확인

확인 항목:

- 월별 폴더가 존재하는지
- `교통사용_내역.xlsx`가 있는지
- `01_근태확인.txt` 또는 이에 준하는 근태 메모가 있는지
- 영수증 이미지 파일명이 날짜 또는 유형을 알아볼 수 있게 되어 있는지
- 전자결재 신규 문서 화면이 Chrome에 열려 있는지

잠금 파일, 임시 파일, 중복 파일은 입력 대상에서 제외한다.

### 3.2 교통사용 내역 분류

우선순위는 다음과 같다.

1. 엑셀 취소선 수동 제외
2. 집/본사 정기 출퇴근 제외
3. 주말 또는 공휴일인데 근태 메모에 출근 근거 없음
4. 연차 또는 반차일 이동
5. 청구 가능 후보

상태값:

| 상태 | 의미 | 브라우저 입력 |
| --- | --- | --- |
| `claimable` | 청구 가능 후보 | 입력 |
| `manual_excluded_strikethrough` | 사용자가 엑셀 취소선으로 제외 | 입력 안 함 |
| `excluded_commute` | 정기 출퇴근 구간 | 입력 안 함 |
| `exception_weekend_no_attendance` | 주말 이동이나 근태 근거 없음 | 입력 안 함 |
| `review_required` | 사용자 확인 필요 | 기본 입력 안 함 |

### 3.3 식대 및 택시 영수증 분류

야근식대:

- 근태 메모에 퇴근 시간이 있는 날짜만 청구 후보로 둔다.
- 전자결재 상세내역에는 퇴근 시간을 적는다.
- 퇴근 시간이 없으면 `review_required`로 둔다.

외근 식대:

- 근태 메모에 외근일로 표시된 날짜만 청구 후보로 둔다.
- 프로젝트 또는 고객사 매핑이 불명확하면 `review_required`로 둔다.

택시:

- 영수증 이미지와 교통사용 내역 또는 사용자 메모가 매칭되는 경우만 청구 후보로 둔다.
- 출발지, 도착지, 사용 사유가 불명확하면 `review_required`로 둔다.

### 3.4 입력 전 미리보기

전자결재 입력 전 아래 구분으로 미리보기를 만든다.

```text
## 전자결재 입력 대상
- 날짜 / 계정항목 / 금액 / 상세내역 / 첨부파일

## 입력 제외
- 날짜 / 계정항목 / 금액 / 제외 사유

## 사용자 검토 필요
- 날짜 / 계정항목 / 금액 / 확인할 내용

## 첨부 대상
- 파일명 / 연결 항목 / 확인 상태
```

미리보기에서 사용자가 확인해야 하는 핵심 항목:

- 입력 대상 건수
- 제외 건수
- 검토 필요 건수
- 총 청구 금액
- 첨부파일 개수
- 프로젝트 코드와 계정과목

### 3.5 브라우저 입력

Chrome 전자결재 화면에 입력할 때의 원칙:

- 문서 유형이 `지출결의서(개인경비)`인지 먼저 확인한다.
- `claimable` 상태의 항목만 입력한다.
- 상세 행이 부족하면 행 추가 후 입력한다.
- 화면 합계와 내부 계산 합계가 일치하는지 확인한다.
- 첨부 전 파일 목록을 사용자에게 보여준다.
- 입력 완료 후 결재요청 직전 상태에서 멈춘다.

### 3.6 상신 후 확인

사용자가 직접 결재요청을 클릭한 뒤 Codex가 확인할 수 있는 항목:

- 현재 URL이 신규 작성 화면이 아닌 제출 문서 조회 화면인지
- 문서 유형이 `지출결의서(개인경비)`인지
- 상신일시가 표시되는지
- 현재 결재 대기자가 누구인지
- 합계 금액이 입력 전 미리보기와 일치하는지
- 첨부파일 개수가 예상과 일치하는지

주의:

- `상신취소` 버튼이 보이면 상신된 문서라는 근거로만 사용한다.
- Codex는 사용자의 명시 지시가 없는 한 `상신취소`, `삭제`, `수정`, `회수` 버튼을 누르지 않는다.

## 4. 공개 저장소에 남기지 않을 정보

이 저장소는 공개될 수 있으므로 아래 정보는 커밋하지 않는다.

- 실제 영수증 이미지
- 실제 교통사용 내역 엑셀
- 실제 근태 메모
- 집/회사 정류장명
- 고객사명과 방문 상세
- 실제 프로젝트 코드
- 그룹웨어 문서 번호
- 결재 금액 합계
- 계정, 쿠키, 토큰, 세션 파일

실제 값은 NAS 월별 폴더, `local/`, `config/*.local.*` 같은 Git 제외 경로에만 둔다.

## 5. 다음 실행 시 Codex에게 줄 지시문

아래 문구를 다음 월초청구서 처리 시 그대로 사용하면 된다.

```text
월초청구서 처리 도와줘.

자료 폴더:
D:\02_nas\onycom\01_월초청구서\YYYY년M월 월초청구서

전자결재 신규 지출결의서 화면은 Chrome에 열어뒀어.
docs/codex-assisted-expense-workflow.md 기준으로 진행해줘.

1. 폴더 파일 목록을 확인해줘.
2. 교통사용_내역.xlsx에서 취소선 표시된 행은 제외해줘.
3. 01_근태확인.txt를 읽고 야근, 외근, 연차, 반차, 주말출근 여부를 반영해줘.
4. 자동 입력 대상, 입력 제외, 사용자 검토 필요 항목을 먼저 미리보기로 작성해줘.
5. 내가 미리보기를 확인하면 Chrome 전자결재 화면에 입력 대상만 입력해줘.
6. 첨부파일 목록은 입력 전 다시 확인해줘.

결재요청은 내가 직접 누를게.
상신 후에는 문서 상태, 현재 결재자, 합계, 첨부파일 개수만 확인해줘.
```

## 6. 구현 방향

초기에는 Codex가 상황을 판단해 보조하고, 반복되는 부분만 Python으로 분리한다.

우선 구현할 자동화:

- 월별 폴더 manifest 생성
- `교통사용_내역.xlsx` 취소선 감지
- 근태 메모 파서
- 출퇴근 제외 구간 local 설정 로더
- 입력 대상/제외/검토 필요 Markdown 미리보기 생성

나중에 구현할 자동화:

- 전자결재 DOM 필드 매핑 자동 점검
- Chrome 입력 dry-run
- 상세 행 자동 입력
- 첨부파일 업로드 보조
- 상신 후 읽기 전용 상태 점검
